ULCL Dynamic
Traffic Steering

This guide demonstrates how to implement Dynamic Traffic Steering in a 5G Core network using the Uplink Classifier (ULCL). With the free5GC framework, it demonstrates how to route user traffic across different Anchor UPFs efficiently, adapting to real-time requirements.

Part of the network topology implementation, we used this guide by s5uishida.
It describes a simple network topology that uses free5GC and UERANSIM for ULCL with one I-UPF and two PSA-UPFs. We used go-upf for the implementation of the UPFs.

free5GC v3.3.0 UERANSIM gtp5g v0.8.6

File 3 — steering.go (Core Logic)

Path: NFs/smf/internal/sbi/processor/steering.go

The key insight: search only non-default data paths for the SDF-filtered PDR. The default path's FAR (FARID=1) is already correct — the ULCL branching PDR lives exclusively on a specificPath.

go processor/steering.go 
package processor

import (
    "fmt"
    "net"
    "net/http"

    "github.com/free5gc/free5gc/NFs/smf/internal/context"
    smf_context "github.com/free5gc/free5gc/NFs/smf/internal/context"
    "github.com/free5gc/free5gc/NFs/smf/internal/pfcp/session"
    "github.com/gin-gonic/gin"
)

type SteerRequest struct {
    SUPI      string `json:"supi"`
    TargetUPF string `json:"targetUpf"`
}

func (p *Processor) HandleSteerTraffic(c *gin.Context) {
    var req SteerRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        c.JSON(http.StatusBadRequest, gin.H{"error": err.Error()})
        return
    }

    // 1. Find the SMContext for this SUPI
    var smContext *smf_context.SMContext
    smf_context.GetSelf().UePool.Range(func(key, value interface{}) bool {
        ue := value.(*smf_context.UEContext)
        ue.RLock()
        defer ue.RUnlock()
        ue.SmContextList.Range(func(k2, v2 interface{}) bool {
            ctx := v2.(*smf_context.SMContext)
            if ctx.Supi == req.SUPI {
                smContext = ctx
                return false
            }
            return true
        })
        return smContext == nil
    })
    if smContext == nil {
        c.JSON(http.StatusNotFound, gin.H{
            "error": "SMContext not found",
            "supi":  req.SUPI,
        })
        return
    }
    smContext.SMLock.Lock()
    defer smContext.SMLock.Unlock()

    // 2. Find target UPF node by IP
    var targetUPNode *smf_context.UPNode
    smf_context.GetSelf().UserPlaneInformation.UPNodes.Range(func(k, v interface{}) bool {
        node := v.(*smf_context.UPNode)
        if node.UPF != nil {
            ip := node.UPF.NodeID.ResolveNodeIdToIp().String()
            if ip == req.TargetUPF {
                targetUPNode = node
                return false
            }
        }
        return true
    })
    if targetUPNode == nil {
        c.JSON(http.StatusNotFound, gin.H{
            "error":     "Target UPF not found",
            "targetUpf": req.TargetUPF,
        })
        return
    }

    targetIP := net.ParseIP(req.TargetUPF)
    if targetIP == nil {
        c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid target UPF IP"})
        return
    }

    // 3. Find the ULCL branching node — SDF-filtered PDR on a NON-DEFAULT path
    var iUPFNode *context.DataPathNode
    for _, dp := range smContext.Tunnel.DataPathPool {
        if dp.IsDefaultPath {
            continue // Skip default path — FARID=1 is already correct
        }
        for node := dp.FirstDPNode; node != nil; node = node.Next() {
            if node.UpLinkTunnel != nil && node.UpLinkTunnel.PDR != nil {
                pdr := node.UpLinkTunnel.PDR
                if pdr.PDI.SDFFilter != nil &&
                    pdr.FAR != nil &&
                    pdr.FAR.ForwardingParameters != nil {
                    iUPFNode = node
                    break
                }
            }
        }
        if iUPFNode != nil {
            break
        }
    }
    if iUPFNode == nil {
        c.JSON(http.StatusInternalServerError, gin.H{
            "error": "I-UPF node not found in data path",
        })
        return
    }

    // 4. Update the uplink FAR to point to the new target UPF
    ulFAR := iUPFNode.UpLinkTunnel.PDR.FAR
    if ulFAR == nil || ulFAR.ForwardingParameters == nil {
        c.JSON(http.StatusInternalServerError, gin.H{
            "error": "Uplink FAR has no forwarding parameters",
        })
        return
    }

    teid := uint32(1)
    if ulFAR.ForwardingParameters.OuterHeaderCreation != nil {
        teid = ulFAR.ForwardingParameters.OuterHeaderCreation.TEID
    }
    ulFAR.ForwardingParameters.OuterHeaderCreation = &context.OuterHeaderCreation{
        Ipv4Address: targetIP.To4(),
        TEID:        teid,
    }
    ulFAR.State = context.RULE_UPDATE

    // 5. Send PFCP Session Modification to I-UPF
    if err := session.SendPfcpSessionModificationRequest(
        iUPFNode.UPF,
        smContext,
        []*context.PDR{iUPFNode.UpLinkTunnel.PDR},
        nil, nil,
    ); err != nil {
        c.JSON(http.StatusInternalServerError, gin.H{
            "error": fmt.Sprintf("PFCP modification failed: %v", err),
        })
        return
    }

    // 6. Commit updated context
    smContext.SelectedUPF = targetUPNode.UPF

    c.JSON(http.StatusOK, gin.H{
        "status":    "steering applied",
        "supi":      req.SUPI,
        "targetUpf": targetUPNode.UPF.NodeID.ResolveNodeIdToIp().String(),
        "targetIp":  req.TargetUPF,
    })
}

Option B: Use steer.sh script (Recommended)

This script automates steering a UE to a target UPF. Usage:

bash
./steer.sh <Targeted PSA-UPF IP>
# Example: steer to PSA-UPF2
./steer.sh 10.160.101.198
steer.sh 
#!/bin/bash
TARGET=$1
SUPI="imsi-208930000000001"

if [ -z "$TARGET" ]; then
    echo "Usage: $0 <target-upf-ip>"
    exit 1
fi

echo "Steering to $TARGET..."
for i in $(seq 1 20); do
    RESULT=$(sudo curl -s -v -X POST http://127.0.0.2:8000/nsmf-steering/v1/steer \
        -H 'Content-Type: application/json' \
        -d "{\"supi\":\"$SUPI\",\"targetUpf\":\"$TARGET\"}" 2>&1)

    FARID=$(echo "$RESULT" | grep -i "x-debug-far" | grep -oP 'FARID=\K\d+')
    echo "Attempt $i: FARID=$FARID"

    # FAR IDs 5 and 7 are the active steering FARs — accept either
    if [[ "$FARID" == "5" || "$FARID" == "7" ]]; then
        echo "SUCCESS on attempt $i — steered to $TARGET"
        exit 0
    fi
    sleep 0.1
done
echo "Failed after 20 attempts"

Troubleshooting

SMContext not found — curl returns 404
The UE is not connected. Reconnect it:
bash 
sudo pkill -f nr-ue
cd ~/UERANSIM/build && sudo ./nr-ue -c ../config/free5gc-ue.yaml &
"No PDR match this skb" in dmesg
The gtp5g kernel patch was not applied or the old module is still loaded. Verify:
bash 
sudo dmesg | grep gtp5g
ls -la /lib/modules/$(uname -r)/kernel/drivers/net/gtp5g.ko
Ensure the patched .ko is in /lib/modules/ and reloaded. Re-apply Section 5 if needed.
I-UPF node not found in data path
The ULCL session was not established. Verify that uerouting.yaml specificPath starts with [I-UPF, PSA-UPF]. Check SMF logs:
bash 
grep -i 'ulcl\|error' /tmp/smf_debug.log | tail -20
PSA-UPF has no PDRs after restart
PFCP association was lost. Always restart all UPFs after restarting SMF. PFCP sessions are re-established on the next UE connection.
"file exists" error on UPF start
The upfgtp interface is still up from a previous run:
bash 
sudo ip link delete upfgtp
"address already in use" on SMF start
An old SMF process is still running:
bash 
sudo pkill -f 'bin/smf' && sleep 2
GTP packets arrive at I-UPF but not PSA-UPF upfgtp
Packets are being dropped in the kernel — the gtp5g patch is missing on PSA-UPF. Apply Section 5 on PSA-UPF and rebuild.
Steering API updates the wrong FAR (FARID=1)
The code is searching the default path. Ensure the IsDefaultPath check is present in steering.go — the loop must continue (skip) any path where dp.IsDefaultPath == true.
